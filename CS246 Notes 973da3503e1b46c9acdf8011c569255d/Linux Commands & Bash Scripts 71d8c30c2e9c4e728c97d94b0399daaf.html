Linux Commands

Command: ls

List all contents in the current directory (excluding hidden files)

Command: ls -a

List all contents of the current directory (including hidden files)

The first “.” represents the current directory.

The “..” represents the parent directory.

Command: ls -l

List all contents of current directory in long format.

Command: cd (change directory)

cd: go to default / home directory

cd ~: go to default / home directory

cd path: follow the path

cd ..: go to the parent directory

cd -: go to the previous directory

Command: mkdir dirName

Create new directory named dirName in the current directory.

Command: rmdir dirName

Remove directory dirName in the current directory, only works if dirName is empty.

Command: rm filename

Remove filename in the current directory.

Command: cp filename1 filename2

Copy contents in filename1 and paste them into filename2. Contents in filename2 will be overwritten.

Command: mv filename1 filename2

Cut filename1 and paste its contents in filename2. filename1 does not exist any more. This command is usually used to re-name files.

Command: mv filename1 dir

Move filename1 under dir.

Linux Stream

stdin: standard input

stdout: standard output (the first output stream [1] )

stderr: standard error (the second output stream [2] )

Note:

stdout is not real-time. It’s buffered. stderr is real-time.

Stream Redirection Symbol: >&

Syntax for redirecting a stream to another stream.

stdin: 0

stdout: 1

stderr: 2

Example:

2>&1: Change stderr to stdout. Now all output written to stderr will be written to stdout.

1>&2: Similarly, change stdout to stderr. Now all output written to stdout will be written to stderr.

Wildcard Matching (Part 1)

Globbing patterns:

  • The standard wildcard matching.
  • Used by various command-line utilities to work with multiple files.
  • Pattern: a string that contains * or ? or [ ]. (appears as an argument of command)
  • Globbing: an operator that expends the pattern into a list of the path names matching the pattern.

Patterns:

? : matches any single character

* : matches any string (with any length)

[…] : matches any one of the characters inside the brackets.

[… - …] : matches any one of the characters within the range of characters in the brackets.

[!…] : matches any character that is NOT inside the brackets.

[!… - …] : matches any character that is NOT within the range of characters in the brackets.

{ } : recognize multiple patterns. (separated by comma ,) NO WHITESPACES!!!

(i.e.: {pattern1,pattern2})

Note:

  • * and ? must be used OUTSIDE the brackets [ ] to perform the correct matching.
  • And they do not match /.
  • [… - …] denotes a range (in ASCII order) & left and right cannot be empty.

Examples:

(1) Write a pattern that contains the file extension .txt at the end.

Answer: *.txt

(2) Write a pattern that ends with string “ab” with a string length of 3.

Answer: ?ab

(3) Write a pattern that starts with character ‘a’.

Answer: a*

(4) Write a pattern that starts with character ‘a’ and ends with file extension .txt.

Answer: a*.txt

(5) Write a pattern that contain a single digit (0 to 9) at the end.

Answer: *[0123456789] OR *[0-9]

(6) Write a pattern that does not contain a single digit (0 to 9) at the end.

Answer: *[!0123456789] OR *[!0-9]

(7)* Tricky question 1: What does the pattern [][!] represent?

Answer: matching ], [, and ! (brackets cannot be empty!)

(8)* Tricky question 2: What does the pattern [1-] represent?

Answer: matching 1 and -.

(9) Write a pattern that starts with an uppercase letter and ends with either a uppercase or lowercase letter.

Answer: [A-Z]*[A-Za-z] ([ ] can have multiple ranges)

Wildcard Matching (Part 2)

We can use globbing patterns to deal with arbitrary number of files by writing patterns that match all the filenames, can we use patterns to deal with the contents in a file?

Answer: YES!

Command: egrep [pattern] [paths to file(s)]

  • Output lines that contain a match for the pattern
  • Be able to handle multiple patterns, as long as there exists a match for one pattern, the corresponding line will be displayed.
  • Corresponding lines will be displayed.

File Permissions

Command: chmod

  • u: user
  • g: group
  • o: other
  • a: all
  • +: add permission
  • -: remove permission
  • r: read
  • w: write
  • x: execute

Syntax:

chmod [target] [+/-] [permission] [filename]

Examples:

  • Allow group to read : chmod g+r in.txt
  • Allow others to execute : chmod o+x in.txt
  • Deny user to write : chmod u-w in.txt
    • Deny everyone to execute : chmod a-x in.txt

Shell Variables

Variable definition: x=5 (NO whitespace!!!)

Note:

  • Shell variable holds strings, without ‘$’, variable is just a string

    e.g.: echo x will just display “x”

  • Double quotes allow retrieving variables

    e.g.: echo “${x}” will display the value of x

  • Single quotes does not allow retrieving variables

    echo ‘${x}’ will just display ${x}

Bash/Shell Scripts

Test files containing Linux commands executed as a program.

2 methods to receive input data from user:

  1. command line arguments
  1. standard input

Command: #!/bin/bash

This line is telling the interpreter that the following lines are written for bash, so the file will be executed as bash script.

Arguments:

  • $0: represents the script itself
  • $1: represents the first argument
  • $2: represents the second argument
  • same for $3, $4, …
  • $#: represents the number of arguments inputted
  • $?: represents the exit status of previous Linux command
    • 0: represents successful/true
    • 1: represents unsuccessful/false
  • $@: represents an array of arguments

    e.g.:

    for i in "${@}"; do
	echo ${i}
done

Integer comparison:

  • -eq: equals to
  • -lt: less than
  • -le: less than or equal to
  • -gt: greater than
  • -ge: greater than or equal to
  • -ne: not equal to

String Comparison:

Comparing ASCII values (<, <=, >, >=, ==, !=)

Whitespaces:

Used for if statements:

Syntax:

if [ ${#} -lt 1 ]; then
	echo "hello"
elif [ ${#} -eq 1 ]; then
	echo "world"
else
	echo "apple"
fi

Other Bash keywords:

  • -e: returns whether the file exists

    e.g.: if [ file.txt -e ]; then

  • -r: returns whether the file is readable

    e.g.: if [ file.in -r ]; then

  • -x: returns whether the file is executable

    e.g.: if [ file.cc -x ]; then

  • -w: returns whether the file is writable

    e.g.: if [ file.out -w ]; then

Command embedded:

Syntax: $(command)

Example: $(cat "$1") : return the result from command cat “$1”

Loops in Bash:

Syntax:

## Type 1
for identifier in $(cat "$1"); do
	...
done

## Type 2
for i in {0..9}; do; OR 0 1 2 3 4 (with whitespaces)
	...
done

## Type 3
while [ ${i} -lt 0 ]; do
	...
done

Temp files:

Command for making temp files: mktemp

Example: x=$(mktemp)

x is now a string that stores the ABSOLUTE PATH of the temp file created.

Command for removing the created temp files: rm

Content discarding:

Command: output direction to /dev/null

Example:

cmp $1 $2 >/dev/null: compares the first and second argument, and discard the result to the “garbage can”.

Reason:

Sometimes, we only want the status of the last command instead of the exact result. We don’t really care about the outputs. Therefore, we simply redirect the result to /dev/null and use command $? to get the status of the previous command result.

			echo "ERROR: file ${file}.args is not readable." 1>&2

					echo "ERROR: file ${file}.in is not readable." 1>&2
					exit 1
				else

					./$2 $(cat ${file}.args) <${file}.in >${temp}
				fi
			else
				./$2 $(cat ${file}.args) >${temp}
			fi
		fi

		var_exist=0
		if [ -e ${file}.in ]; then
			if [ ! -r ${file}.in ]; then

				exit 1
			else
				in_exist=1
				./$2 <${file}.in >${temp}
			fi
		else
			./$2 >${temp}
		fi
	fi
	
	if [ ! -e ${file}.out ]; then
		echo "ERROR: ${file}.out file does not exist." 1>&2
