Default Arguments

Definition: A default argument is a value provided in a function declaration/definition that is automatically assigned by the compiler if the caller of the function does not provide a value for the argument with a default value.

Example:

#include <iostream>

int foo(int x, int y = 2) {
	return x + y;
}

int main() {
	std::cout << foo(1, 3) << std::endl;
	// pass in arguments: x=1 and y=3, return 4
	std::cout << foo(1) << std::endl;
	// pass in only x=1, then foo is run with default argument y=2, return 3
}

Note:

  • Once a default argument appears in a function’s declaration, then all subsequent arguments must be default arguments. (In other words, all default arguments should only appear at the end.)

For example:

int foo(int x, int y = 2); → this is correct

int foo(int x = 2, int y); → this is incorrect and will cause an error (failure during compilation)

This error occurs because when we’re calling foo(2), the compiler doesn’t know that should 2 be assigned to x or y.

  • Compiler will know the difference by the number of values passing or by the type of parameters.
  • Compiler recognizes functions by function signatures.

Function Signatures

  • Compiler recognizes functions by function signatures.
  • Includes the function name, and number/order/type of its parameters.
  • Does not include return type.

Example:

void goo(int x, char c = ’a’, std::string str = "CS")；

Three function signatures are generated:

goo(int);

goo(int, char);

goo(int, char, std::string);

Note: We must not write any other functions whose signatures match any of the above three signatures!

Unambiguity:

  • Compiler is not smart enough when ambiguity of function signature occurs, thus error will occur at compiling time.
  • Make sure that our code has the unambiguity of function signatures.
  • We can have functions with the same name in C++, as long as they don’t have the same function signatures. (Discussed in the following section)

Function Overloading

  • Two or more functions that share the same name with different parameters (different function signatures)
  • Considered as polymorphism feature in C++ (will be taught later in this course)
  • Not allowed in C

Example:

#include <iostream>

void goo(int x, char c = 'a') { }
void goo(int x, int y) { }

int main() {
	goo(5, 5);
	goo(5);
}
// This is correct because their function signatures are different

Structure

Structure definition in C vs in C++

Structure definition in C

struct student {
	int id;
	int age;
	char gender;
}; // must have a semicolon

// Method 1
int main(void) {
	// without field initialization
	struct student a;
	// with field initialization
	struct student b = {123, 12, 'M'};
}

// Method 2
typedef struct student student;
// omit prefix struct in C
int main(void) {
	// without field initialization
	student a;
	// with field initialization
	student b = {123, 12, 'M'};
}

Structure definition in C++

struct student {
	int id;
	int age;
	char gender;
}; // same as C: must have a semicolon

int main() {
	// without field initialization
	student a;
	// with field initialization
	student b = {123, 12, 'M'};
}
  • We don’t need typedef to omit the prefix struct
  • Always need to add the semicolon at the end for structure definition.

Parameter Passing

Passing by value:

  • make a copy of the argument
  • pass the copied value to the called function

Passing by Address:

  • Passing by value is not a good idea if the value passed costs too much stack memory
  • Address is always 8 bytes
  • We can pass by address to avoid copy of the passing value

Passing by Reference:

Syntax:

  • Variable type followed by ampersand &
  • Reference is an alias, another name of the existing variable/object

Example:

int main() {
	int a = 5;
	int& b = a;
}
// b is a reference of a
// we can even say that b is a

Example:

void foo(student& a) { // define that the type of parameter is a reference of student
	a.id = 2005;
	a.age = 30;
	std::cout << "address of a: " << &a << std::endl;
}

int main() {
	student a = {123, 12, 'M'};
	student& s = a;
	std::cout << "address of s: " << &s << std::endl;
	foo(a); // only pass the reference of s to function foo
}
  • function foo takes reference of student as parameter
  • we call this process passing by reference (avoid copy)
  • a is mutated in function foo, which means that s is mutated since a is an alias of s
  • s and a have the same address

Reference vs Pointer

Pointers:

  • can be NULL
  • can be initialized at anytime
  • can point to other objects
  • needed to be dereferenced

References:

  • cannot be NULL
  • must be initialized when created
  • once initialized, cannot be changed
  • automatic dereferencing
  • act as const pointer, but is NOT const pointer

Note:

  • When passing by pointers pointing to structures, to avoid dereferencing, we need to use arrow -> to access or mutate fields.
  • When passing by reference , we can just use . to access or mutate fields.

Lvalue & Rvalue

Lvalue:

  • has memory location that we can access.
  • “anything” that can appear on the left-hand side of assignment operator ‘=’.
  • if passing by reference (not constant reference), then the reference MUST BE lvalue.
  • if passing by const reference, then reference CAN be rvalue.

Rvalue:

  • has memory location, but we cannot access.
  • if returned by value, then returned values from functions are ALWAYS rvalues.

Example 1:

Error:

  • passing by reference must be lvalue, but 5 is a rvalue.
  • resulting in a compiling error.

Example 2:

Error:

  • 12 is a rvalue, contradicts with lvalue definition.
  • resulting in a compiling error.

Example 3: lvalue-to-rvalue convertions

  • Both x and y are lvalues (have memory locations).
  • Operand + requires rvalues and returns a rvalue.
  • x and y are converted to rvalues.

Note:

lvalues can be converted to rvalues, but rvalues can NEVER be converted to lvalues!

Example 4: constant reference

  • function foo takes a constant reference to integer
  • it promises not to mutate the integer (literal reference)
  • rvalue result is located at a temporary memory location
  • then rvalue reference is acceptable as parameter of foo
  • only works for constant references!

Note:

const int& n = 5; is almost equivalent to const n = 5;

IMPORTANT: if the parameter type is const reference, then rvalue is acceptable. (only under this condition!)

Summary of Lvalue & Rvalue:

  • lvalue reference must be initialized with a lvalue
  • cannot create a reference to a reference
  • cannot create a pointer to a reference
  • cannot create an array of reference
  • pass arguments as references for anything bigger than integers

Side notes:

  • a++ returns a rvalue reference (return its original value first, then augment by 1)
  • ++a returns a lvalue reference (augment by 1 first, then return its value)

Operator Overloading

Some operators serve for more than one purpose. They can do different things under different circumstances.

Example:

  • We can use operator + to add integers
  • We can also use operator + to add two strings
  • This is called operator overloading
  • Operator overloading allows more than one definition of an operator in the same scope.
  • Compiler decides the most appropriate definition to use.
  • Syntax: the keyword operator followed by the symbol of the operator.
  • Has a return type and a parameter list (often passed by constant reference).

Conditional Compilation

Conditional Compilation directives are type of directives which helps to compile a specific portion of the program or to skip compilation of some specific parts of the program based on some conditions.

Sample main.cc:

#include <iostream>

#ifndef VALUE // will be executed if VALUE if not defined
#define VALUE 5
#endif

#define NUM 100 // define NUM with value 100
// no assignment operator, no semicolon

int main() {
	int a = VALUE + 3 + 4;
	
	#ifdef NUM // will be executed if NUM if defined
	std::cout << a << std:: endl;
	#endif
}

Linux command: g++14 -D VALUE=10 main.cc, which is equivalent to #define VALUE 10

Compiling

Question: If our program consists only one file, then we only need to compile one file, but what should we do if our program consists of more than one file?

For example:

calculateLine.h

sum.cc

#ifndef __CALCULATELINE_H__
#deinfe __CALCULATELINE_H__

#include <string>

void calculateLine(std::string line);

#endif
#include <iostream>
#include "calculateLine.h"

int main() {
	std::string line;
	while (std::geline(std::cin, line)) {
		calculateLine(line);
	}
}

calculateLine.cc

#include <iostream>
#include <sstream>
#include "calculateLine.h"

void calculateLine(std::string line) {
	std::istringstream iss{line};
	int num, sum;
	char op;
	iss >> sum;
	while (iss >> op >> num) {
		if (op == '+') {
			sum += num;
		} else {
			sum -= num;
		}
	}
	std::cout << sum << std::endl;
}

How do we compile the three files above into a program?

Method 1: Compile manually (use this for debugging)

  • Compile all the .cc files at the same time by executing the following command:

    g++14 sum.cc calculateLine.cc

or g++14 sum.cc calculateLine.cc -o sum (-o renames the generated file into sum)

or g++14 *.cc -o sum (using globbing pattern)

  • Every time we make a change to any file, we need to re-compile everything.
  • If the program is large, this might take a long time to compile.

Method 2: Create object files

  • Create objects depending on file dependencies.
  • calculateLine.cc depends on calculateLine.h, then one object (file extension .o) will be created.
  • sum.cc depends on calculateLine.h, then another object will be created.
  • Compile the two objects to create the executable:
    g++14 -c calculateLine.cc calculateLine.h   (calculateLine.o is created)
g++14 -c sum.cc calculateLine.h           (sum.o is created)
g++14 sum.o calculateLine.o -o sum     (create the executable based on two objects)
  • If making changes, we only need to re-create the object associated.
  • If many files are changed, it would be hard to keep track of objects and it would be better to just re-compile everything again.
  • Still too complicated!

Method 3: Makefile

Makefile

Linux command: make

  • a Unix tool to simplify building program executables from arbitrary number of modules.
  • automatically update objects that need to be re-built since last time.
  • must have a file named “Makefile” under the same directory.
  • we will need to write “Makefile” manually.
  • type Linux command make will then automatically compile the program based on what we write in Makefile.

Makefile template:

CXX = g++
CXXFLAGS = -std=c++14 -Wall
OBJECTS = sum.o calculateLine.o
EXEC = sum

${EXEC}: sum.o calculateLine.o
				${CXX} ${CXXFLAGS} ${OBJECTS} -o ${EXEC}

sum.o: sum.cc calculateLine.h

calculateLine.o: calculateLine.cc calculateLine.h

.PHONY: clean

clean: rm ${OBJECTS} ${EXEC}

Side notes:

  • -Wall: all the warnings will be treated as error messages
  • sum: the generated executable file if make is successful
  • another flag (but not often used here): -g: enables using gdb
  • make clean command: deletes all files created by Makefile
  • IMPORTANT: There are TWO TABS before ${CXX}! (It must be 2 tabs! Otherwise it won’t compile! (”missing tab” error will occur))

UW’s Makefile template (Best version):

CXX = g++
CXXFLAGS = -std=c++14 -Wall -MMD
# -MMD generates the dependencies of each object files
EXEC = exec
SOURCES = $(wildcard *.cc)
OBJECTS = ${SOURCES:.cc=.o}
DEPENDS = ${OBJECTS:.o=.d}

${EXEC}: ${OBJECTS}
	${CXX} ${CXXFLAGS} ${OBJECTS} -o ${EXEC}

-include ${DEPENDS}

#PHONY target
.PHONY: clean
clean:
	rm ${OBJECTS} ${EXEC} ${DEPENDS}

Underlined parts are all we need to modify for each project.
